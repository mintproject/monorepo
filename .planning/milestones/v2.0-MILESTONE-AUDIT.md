### Missing (2/14)

**Gap 1 (CRITICAL): `has_accepted_values` column missing from modelcatalog_parameter**
- **From:** `get-modelcatalog-setup.graphql` line 19, `get-modelcatalog-configuration.graphql` line 19
- **Expected:** Column on `modelcatalog_parameter` table
- **Reality:** Not created in any migration (1771105509000, 1771105510000, 1771105511000, 1771105512000)
- **Impact:** All Ensemble Manager model run fetches fail with Hasura field-not-found error
- **Fix:** Either add `has_accepted_values TEXT[]` column to modelcatalog_parameter migration, or remove the field from both .graphql queries

**Gap 2 (CRITICAL): Wrong FK column name in custom_datasetspecifications_get handler**
- **From:** `custom-handlers.ts` line ~494: `modelcatalog_configuration_input(where: { model_configuration_id: { _eq: $cfgId } })`
- **Expected column:** `model_configuration_id`
- **Actual column:** `configuration_id` (per migration 1771105509000 `modelcatalog_configuration_input` CREATE TABLE)
- **Impact:** `GET /v2.0.0/custom/datasetspecifications?configurationid=X` returns 500 error
- **Fix:** Change `model_configuration_id` to `configuration_id` in custom-handlers.ts filter
