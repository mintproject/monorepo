## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-04 | Tampering | service.ts junctionColumns spread | mitigate | Only columns listed in relConfig.junctionColumns are spread; no arbitrary client key can be passed through to Postgres; boolean type enforced by Hasura schema |
| T-12-04b | Tampering | request.ts buildJunctionInserts junctionColumns spread | mitigate | Same guard: only relConfig.junctionColumns keys are spread onto the outer junction row; client cannot inject arbitrary column names |
| T-12-05 | Information Disclosure | openapi.yaml DatasetSpecification | accept | isOptional is non-PII metadata; no access-control change needed — existing user/anonymous permissions already control read access |
| T-12-06 | Spoofing | is_optional in Hasura insert | mitigate | Apollo Client uses parameterized mutations; no string interpolation path; Hasura's boolean column type rejects non-boolean values at the DB layer |
</threat_model>
