**config.py changes:**

Add 10 new TYPE_ constants matching the RDF types from the ontology:
```python
TYPE_PERSON = f"{SD}Person"
TYPE_MODEL_CATEGORY = f"{SDM}ModelCategory"

TYPE_PROCESS = f"{SDM}Process"

TYPE_IMAGE = f"{SD}Image"
TYPE_VARIABLE_PRESENTATION = f"{SD}VariablePresentation"
TYPE_INTERVENTION = f"{SDM}Intervention"
TYPE_GRID = f"{SDM}Grid"
```

Note: Check the ontology.ttl to confirm exact namespace prefixes (sd: vs sdm:). The ontology defines `sd:Person`, `sd:Image`, `sd:VariablePresentation` under the `sd:` namespace and `sdm:ModelCategory`, `sdm:Region`, `sdm:Process`, `sdm:TimeInterval`, `sdm:CausalDiagram`, `sdm:Intervention`, `sdm:Grid` under the `sdm:` namespace.

**extract.py changes:**

1. Add 10 new extraction functions following the same pattern as existing ones (SPARQL query, iterate results, build dict list):

   - `extract_persons(ds)`: Query `sd:Person` with `sd:name` property. Return list of {id, label, name}.
   - `extract_model_categories(ds)`: Query `sdm:ModelCategory` with optional `sdm:hasModelCategory` for parent self-reference. Return entities + parent_links dict.
   - `extract_regions(ds)`: Query `sdm:Region` with `sd:description`, optional `sdm:partOf` self-reference. Return entities + part_of_links dict.
   - `extract_processes(ds)`: Query `sdm:Process`. Return list of {id, label}.
   - `extract_time_intervals(ds)`: Query `sdm:TimeInterval` with `sdm:intervalValue`, `sdm:intervalUnit`, `sd:description`. Return list.
   - `extract_causal_diagrams(ds)`: Query `sdm:CausalDiagram`. Return list of {id, label}.
   - `extract_images(ds)`: Query `sd:Image` with `sd:description`. Return list.
   - `extract_variable_presentations(ds)`: Query `sd:VariablePresentation` with `sd:hasLongName`, `sd:hasShortName`, `sd:hasStandardVariable`, `sd:usesUnit`. Return list.
   - `extract_interventions(ds)`: Query `sdm:Intervention` with `sd:description`. Return list.
   - `extract_grids(ds)`: Query `sdm:Grid` with `sd:hasDimension`, `sdm:hasShape`, `sdm:hasSpatialResolution`, `sdm:hasCoordinateSystem`, `sdm:gridType`. Return list.

2. Update `extract_software_versions(ds)` to also extract new column values:
   - Add OPTIONAL clauses for `sd:shortDescription`, `sdm:limitations`, `sdm:parameterization`, `sdm:runtimeEstimation`, `sdm:theoreticalBasis`
   - Map to entity dict keys: short_description, limitations, parameterization, runtime_estimation, theoretical_basis

3. Update `extract_model_configurations(ds)` to extract:
   - Add OPTIONAL clause for `sdm:hasModelResultTable`
   - Map to entity dict key: has_model_result_table

4. Update `extract_model_configuration_setups(ds)` to extract new column values:
   - Add OPTIONAL clauses for `sd:author` (single-valued FK), `sdm:calibrationInterval`, `sdm:calibrationMethod`, `sdm:parameterAssignmentMethod`, `sdm:validUntil`
   - Map to entity dict keys: author_id (URI of Person), calibration_interval, calibration_method, parameter_assignment_method, valid_until

5. Add link extraction queries for 14 new junction tables. Extract these as separate link dicts inside the relevant entity extraction functions or as standalone queries in extract_all:

   **SoftwareVersion level (6 link sets):**
   - `sdm:hasModelCategory` on SoftwareVersion -> ModelCategory (for software_version_category)
   - `sdm:hasProcess` on SoftwareVersion -> Process (for software_version_process)
   - `sdm:hasGrid` on SoftwareVersion -> Grid (for software_version_grid)
   - `sdm:hasExplanationDiagram` on SoftwareVersion -> Image (for software_version_image)
   - `sdm:hasInputVariable` on SoftwareVersion -> VariablePresentation (for software_version_input_variable)
   - `sdm:hasOutputVariable` on SoftwareVersion -> VariablePresentation (for software_version_output_variable)

   **Configuration level (3 link sets):**
   - `sdm:hasCausalDiagram` on ModelConfiguration -> CausalDiagram (for configuration_causal_diagram)
   - `sdm:hasOutputTimeInterval` on ModelConfiguration -> TimeInterval (for configuration_time_interval)
   - `sdm:hasRegion` on ModelConfiguration -> Region (for configuration_region)

   **Setup level (3 link sets):**
   - `sd:author` on ModelConfigurationSetup -> Person (for setup_author junction, multi-valued)
   - `sdm:calibratedVariable` on ModelConfigurationSetup -> VariablePresentation (for setup_calibrated_variable)
   - `sdm:calibrationTargetVariable` on ModelConfigurationSetup -> VariablePresentation (for setup_calibration_target)

   **Parameter level (1 link set):**
   - `sdm:relevantForIntervention` on Parameter -> Intervention (for parameter_intervention)

   **CausalDiagram level (1 polymorphic link set):**
   - `sdm:hasDiagramPart` on CausalDiagram -> VariablePresentation|Process (for diagram_part with part_type discriminator). To determine part_type, check if the target entity exists in variable_presentations (type='variable') or processes (type='process').

6. Update `extract_all(trig_path)` to call all new extraction functions and return a comprehensive dict with all entity lists and all link dicts. Structure the return value so that new entities appear under keys matching their table names (e.g., 'persons', 'model_categories', 'regions', etc.) and new links appear under 'links' with descriptive keys (e.g., 'version_to_category', 'version_to_process', 'config_to_causal_diagram', 'setup_to_author', 'param_to_intervention', 'diagram_to_part').

Important: The existing extraction functions must remain backward-compatible. Extend them (add OPTIONAL SPARQL clauses and new dict keys) rather than replacing them. The existing link extraction for version_links, configuration_links, setup_links must still work as before.

   **Parameter level:**
