**hasura.py** must implement a `HasuraBackend` class that provides the same interface the 47 API handlers expect from `query_manager`. The API handlers call these methods:
- `query_manager.get_resource(...)` -- handles GET list and GET by id
- `query_manager.delete_resource(...)` -- handles DELETE
- `query_manager.put_resource(...)` -- handles PUT
- `query_manager.post_resource(...)` -- handles POST

For Phase 2 scope, implement `get_resource()` fully. For `delete_resource`, `put_resource`, and `post_resource`, raise `NotImplementedError("Write operations not yet migrated to Hasura")` -- these are out of scope per research recommendations (read-only migration first).

**HasuraBackend.get_resource()** must:
1. Accept the same kwargs as the existing `query_manager.get_resource()`: `id`, `username`, `label`, `page`, `per_page`, `rdf_type_uri`, `rdf_type_name`, `kls`, `custom_query_name`
2. Map `rdf_type_name` to the corresponding `modelcatalog_*` Hasura table using a TYPE_TO_TABLE dict. Build this mapping from the existing vars.py -- e.g., `"ModelConfigurationSetup" -> "modelcatalog_model_configuration_setup"`, `"Model" -> "modelcatalog_software"` (Model maps to Software in the new schema), `"SoftwareVersion" -> "modelcatalog_software_version"`, etc. For entity types with no PostgreSQL table (e.g., Emulator, HybridModel, CoupledModel, Equation, etc.), return empty list for get-all or None for get-one.
3. If `id` is in kwargs, reconstruct full URI: use `build_instance_uri()` logic from `utils/request.py` -- if not a URL, prepend `ENDPOINT_RESOURCE_PREFIX` (env var `PREFIX`). Then query `WHERE id = $full_uri`. If `custom_query_name` is present, also handle it (custom queries search by label or variable -- for custom queries, use `_ilike` matching on label or fall back to a broad search).
4. If no `id`, query all records with optional `label` filter using `_ilike`, respecting `page` and `per_page` via `offset` and `limit` in the GraphQL query.
5. Use the `gql` library with `RequestsHTTPTransport` for sync HTTP calls to Hasura.
6. Use `X-Hasura-Admin-Secret` header (from `HASURA_SECRET` env var) for service-to-service auth.

**Critical: _row_to_pydantic() mapping function** must:
- Wrap every scalar string column value in a list: `row["label"]` -> `["label_value"]`. This preserves the `Optional[List[str]]` contract of every Pydantic model field.
- The `id` field is the ONE exception -- it is `Optional[str]`, NOT wrapped in a list.
- If a value is None, keep it as None (do not convert to empty list).
- For nested object relationships (junction tables), use Hasura nested queries. The GraphQL query must include subfield selections for relationships. Each Pydantic model that has nested objects (e.g., `has_input: Optional[List[DatasetSpecification]]`) expects a list of full Pydantic sub-objects, not just IDs. Query through junction table relationships using the names from tables.yaml: `inputs { input { id label ... } }`, `outputs { output { id label ... } }`, `parameters { parameter { id label ... } }`, etc. Recursively apply the list-wrapping rule to nested object fields.
- Add the `type` field as a list containing the `rdf_type_uri` for each entity (e.g., `["https://w3id.org/okn/o/sdm#ModelConfigurationSetup"]`).

**GraphQL field fragments:** Pre-define field selection strings for each table covering all columns that map to Pydantic model properties. Start with the most critical types:
- `modelcatalog_model_configuration_setup` (most complex, used by Ensemble Manager -- includes inputs, outputs, parameters, author, regions, etc.)
- `modelcatalog_model_configuration` (includes inputs, outputs, parameters)
- `modelcatalog_software` and `modelcatalog_software_version`
For entity types not in PostgreSQL, the TYPE_TO_TABLE lookup returns None and the method returns [] or None.

```python
import os
HASURA_ENDPOINT = os.environ.get("HASURA_ENDPOINT")
if HASURA_ENDPOINT:
    from openapi_server.backends.hasura import HasuraBackend
    query_manager = HasuraBackend(
        endpoint=HASURA_ENDPOINT,
        secret=os.environ.get("HASURA_SECRET", ""),
        resource_prefix=os.environ.get("PREFIX", "https://w3id.org/okn/i/mint/")
    )
else:
    from obasparql import QueryManager
    from openapi_server.settings import (ENDPOINT, ENDPOINT_GRAPH_BASE,
        ENDPOINT_PASSWORD, ENDPOINT_RESOURCE_PREFIX, ENDPOINT_USERNAME,
        CONTEXT_DIRECTORY, QUERY_DIRECTORY)
    query_manager = QueryManager(
        endpoint=ENDPOINT, endpoint_username=ENDPOINT_USERNAME,
        endpoint_password=ENDPOINT_PASSWORD, queries_dir=QUERY_DIRECTORY,
        context_dir=CONTEXT_DIRECTORY, named_graph_base=ENDPOINT_GRAPH_BASE,
        uri_prefix=ENDPOINT_RESOURCE_PREFIX)
```
This preserves backward compatibility -- without HASURA_ENDPOINT, the original SPARQL backend is used.

**Test approach:** These tests validate the STRUCTURE of responses, not exact values (values depend on loaded data). They verify:
1. List endpoints return a Python list

3. All scalar fields in the response are wrapped in lists (the array-of-scalars contract)

**Test cases to implement:**

```python
import pytest
from openapi_server.backends.hasura import HasuraBackend, _row_to_pydantic

# Unit tests for _row_to_pydantic (no Hasura connection needed)
class TestRowToPydantic:
    def test_scalar_fields_wrapped_in_list(self):
        """Every scalar string field must be wrapped in a list per API contract."""
        from openapi_server.models.model_configuration_setup import ModelConfigurationSetup
        row = {"id": "https://w3id.org/okn/i/mint/test_setup", "label": "Test Setup", "description": "A test"}
        result = _row_to_pydantic(row, ModelConfigurationSetup, "modelcatalog_model_configuration_setup")
        assert result.id == "https://w3id.org/okn/i/mint/test_setup"  # NOT a list
        assert result.label == ["Test Setup"]  # wrapped in list
        assert result.description == ["A test"]  # wrapped in list

    def test_none_values_stay_none(self):
        """None/missing fields must remain None, not become empty lists."""
        from openapi_server.models.model_configuration_setup import ModelConfigurationSetup
        row = {"id": "https://w3id.org/okn/i/mint/test_setup", "label": "Test"}
        result = _row_to_pydantic(row, ModelConfigurationSetup, "modelcatalog_model_configuration_setup")
        assert result.description is None

    def test_nested_inputs_mapped(self):
        """Junction table results must be mapped to nested Pydantic objects."""
        from openapi_server.models.model_configuration_setup import ModelConfigurationSetup
        row = {
            "id": "https://w3id.org/okn/i/mint/test_setup",
            "label": "Test",
            "inputs": [{"input": {"id": "https://w3id.org/okn/i/mint/input1", "label": "Input 1", "has_format": "csv"}}],
            "outputs": [],
            "parameters": []
        }
        result = _row_to_pydantic(row, ModelConfigurationSetup, "modelcatalog_model_configuration_setup")
        assert result.has_input is not None
        assert len(result.has_input) == 1
        assert result.has_input[0].id == "https://w3id.org/okn/i/mint/input1"
        assert result.has_input[0].label == ["Input 1"]

    def test_type_field_added(self):
        """The type field must contain the RDF type URI as a list."""
        from openapi_server.models.model_configuration_setup import ModelConfigurationSetup
        row = {"id": "https://w3id.org/okn/i/mint/test_setup", "label": "Test"}
        result = _row_to_pydantic(row, ModelConfigurationSetup, "modelcatalog_model_configuration_setup")
        assert result.type is not None
        assert isinstance(result.type, list)
```
