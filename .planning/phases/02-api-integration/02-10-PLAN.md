<objective>
Add a `type` TEXT column to `modelcatalog_software` and populate it with the RDF type URI from the TriG data.

Purpose: The service.ts `getSoftwareTypeFilter()` and custom-handlers.ts hardcode `type: { _eq: "..." }` in Hasura WHERE clauses, but `modelcatalog_software` has no `type` column. This causes a runtime error: `field 'type' not found in type: 'modelcatalog_software_bool_exp'`. The TriG data has subtype information: out of ~187 entities typed `sdm:Model`, 40 are also `Theory-GuidedModel`, 25 `EmpiricalModel`, 14 `CoupledModel`. The ETL currently discards this type data.

The fix requires three coordinated changes:
1. **Schema**: Add `type TEXT` column to `modelcatalog_software` via Hasura migration
2. **ETL**: Extract the most specific `rdf:type` from TriG and store it in the `type` column
3. **API**: Ensure existing type filtering code works (it already constructs the right WHERE clauses; it just needs the column to exist)

Output: Schema migration, ETL update, field map update. The existing service.ts filtering logic (`getSoftwareTypeFilter`) already produces the correct type URIs.
</objective>

```python
    # Extract rdf:type for software subtype classification
    type_query = f"""
    PREFIX sdm: <{config.SDM}>
    PREFIX rdf: <{config.RDF}>

    SELECT DISTINCT ?id ?rdftype
    WHERE {{
        ?id a <{config.TYPE_SOFTWARE}> .
        ?id rdf:type ?rdftype .
        FILTER(?rdftype != <{config.TYPE_SOFTWARE}>)
        FILTER(STRSTARTS(STR(?rdftype), "{config.SDM}"))
    }}
    """

    # Map entity ID -> most specific subtype URI
    subtype_map = {}
    for row in ds.query(type_query):
        entity_id = str(row.id)
        rdf_type = str(row.rdftype)
        # Keep the most specific subtype (not sdm:Model itself, which is the base)
        if entity_id not in subtype_map or rdf_type != f"{config.SDM}Model":
            subtype_map[entity_id] = rdf_type

    # Set type on each entity
    for entity in results:
        entity['type'] = subtype_map.get(entity['id'], f"{config.SDM}Model")
```
