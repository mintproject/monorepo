**The key insight:** fastify-openapi-glue calls `service[operationId](req, reply)` for each request. The operationIds follow the pattern `{resourcePlural}_{method}` (e.g., `softwares_get`, `softwares_id_get`, `softwares_post`, `softwares_id_put`, `softwares_id_delete`). We can use a JavaScript Proxy to intercept any method call, parse the operationId to extract the resource name and operation, then dispatch to a generic handler.

Create `src/service.ts`:

```typescript
import { RESOURCE_REGISTRY, getResourceConfig } from './mappers/resource-registry'
import { transformRow, transformList } from './mappers/response'
import { toHasuraInput } from './mappers/request'
import { readClient, getWriteClient, gql } from './hasura/client'

class CatalogServiceImpl {
  // Generic list handler: GET /resources?username=X&label=Y&page=N&per_page=M
  async list(resourcePath: string, req: any, reply: any) { ... }

  // Generic get-by-id handler: GET /resources/{id}
  async getById(resourcePath: string, req: any, reply: any) { ... }

  // Generic create handler: POST /resources
  async create(resourcePath: string, req: any, reply: any) { ... }

  // Generic update handler: PUT /resources/{id}
  async update(resourcePath: string, req: any, reply: any) { ... }

  // Generic delete handler: DELETE /resources/{id}
  async deleteResource(resourcePath: string, req: any, reply: any) { ... }
}
```

**List handler** logic:
1. Look up resource config from registry
2. If `hasuraTable` is null, return 501 (not implemented yet)
3. Build a GraphQL query dynamically: `query { modelcatalog_<table>(where: { ... }, limit: $per_page, offset: ($page-1)*$per_page) { <all columns + relationships> } }`
4. Use `readClient.query({ query: gql\`...\`, variables: {...} })` to execute the query via Apollo Client
5. Transform results with `transformList()`
6. **Username filtering (per user decision):** When `username` query param is provided, add `where: { user_id: { _eq: $username } }` to the GraphQL query. The modelcatalog tables have a `user_id` column that stores the username/email of the creator. This implements the locked decision "Keep username query parameter for scoping". If both `username` and `label` are provided, combine them with `_and`.
7. Handle `label` query param: add `where: { label: { _eq: $label } }` filter if provided

**Get-by-id handler** logic:
1. Decode the id from URL params: `decodeURIComponent(req.params.id)`
2. Use Apollo Client: `readClient.query({ query: gql\`query { modelcatalog_<table>_by_pk(id: $id) { <all columns + relationships> } }\`, variables: { id } })`
3. If result is null, reply 404
4. Transform with `transformRow()`

**Create handler** logic:
1. Transform request body with `toHasuraInput()`
2. Generate a URI-based ID if not provided: `https://w3id.org/okn/i/mint/<uuid>`
3. Use `getWriteClient(req.headers.authorization).mutate({ mutation: gql\`...\`, variables: {...} })` to execute an insert mutation via Apollo Client
4. Transform result and reply 201

**Update handler** logic:
1. Decode id, transform body
2. Execute update mutation via getWriteClient
3. Reply 200 with updated resource

**Delete handler** logic:
1. Decode id
2. Execute delete mutation via getWriteClient
3. Reply 202

**Proxy creation:**
```typescript
export const CatalogService = new Proxy(new CatalogServiceImpl(), {
  get(target, prop: string) {
    // Parse operationId: softwares_get -> resource=softwares, op=get
    // softwares_id_get -> resource=softwares, op=id_get
    // custom_model_index_get -> handle custom endpoints separately
    if (prop.startsWith('custom_')) {
      return (req: any, reply: any) => target.handleCustom(prop, req, reply)
    }
    const match = prop.match(/^(.+?)_(id_)?(get|post|put|delete)$/)
    if (match) {
      const [, resource, hasId, method] = match
      if (method === 'get' && hasId) return (req: any, reply: any) => target.getById(resource, req, reply)
      if (method === 'get') return (req: any, reply: any) => target.list(resource, req, reply)
      if (method === 'post') return (req: any, reply: any) => target.create(resource, req, reply)
      if (method === 'put') return (req: any, reply: any) => target.update(resource, req, reply)
      if (method === 'delete') return (req: any, reply: any) => target.deleteResource(resource, req, reply)
    }
    // user_login_post -> special handler
    if (prop === 'user_login_post') {
      return (req: any, reply: any) => target.userLogin(req, reply)
    }
    return (target as any)[prop]
  }
})
```

**security.ts:**
fastify-openapi-glue supports a `securityHandlers` object. The OpenAPI spec uses `BearerAuth` as the security scheme name. Create:

```typescript
export const SecurityHandler = {
  async BearerAuth(req: any, reply: any, params: any[]) {
    const authHeader = req.headers.authorization
    if (!authHeader || !authHeader.startsWith('Bearer ')) {
      reply.code(401).send({ error: 'Missing or invalid Authorization header' })
      return
    }
    // Store the token on the request for later use by service handlers
    req.bearerToken = authHeader.replace('Bearer ', '')
    // Note: We do NOT validate the JWT here -- Hasura validates it.
    // The token is forwarded to Hasura which performs the actual auth check.
  }
}
```
