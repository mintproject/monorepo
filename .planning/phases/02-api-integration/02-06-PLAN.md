  <action>
Read the existing FastAPI custom endpoint implementations to understand what each returns:
- `/Users/mosorio/repos/mint/model-catalog-fastapi/src/openapi_server/apis/model_api.py` (custom model endpoints)
- `/Users/mosorio/repos/mint/model-catalog-fastapi/src/openapi_server/apis/model_configuration_api.py` (custom config endpoints)
- `/Users/mosorio/repos/mint/model-catalog-fastapi/src/openapi_server/apis/model_configuration_setup_api.py` (custom setup endpoints)
- `/Users/mosorio/repos/mint/model-catalog-fastapi/src/openapi_server/apis/dataset_specification_api.py` (custom dataset spec endpoints)

Create `src/custom-handlers.ts` with a handler for each of the 13 custom endpoints:

1. **`/custom/model/index`** (`custom_model_index_get`): List all models (Software) with their versions, configurations, and setup counts. Uses nested Apollo Client query on `modelcatalog_software` with `versions { configurations { setups { id } } }`. Accepts `username` and `label` query params. Filter by `user_id` when `username` is provided (per user decision).

2. **`/custom/model/intervention`** (`custom_model_intervention_get`): List models that have interventions associated with their parameters. Query `modelcatalog_software` joining through versions -> configurations -> setups -> parameters -> interventions.

3. **`/custom/model/region`** (`custom_model_region_get`): List models that operate in specific regions. Query through configurations -> regions.

4. **`/custom/models/variable`** (`custom_models_variable_get`): Find models by variable. Query `modelcatalog_software` joining versions -> input/output variables, filtering by variable label or ID.

5. **`/custom/modelconfigurationsetups/variable`** (`custom_modelconfigurationsetups_variable_get`): Find setups by variable. Query `modelcatalog_model_configuration_setup` joining inputs/outputs -> dataset_specifications, filtered by variable.

6. **`/custom/configurationsetups/{id}`** (`custom_configurationsetups_id_get`): Get a single setup with all nested data (full expansion). Deep query with all relationships expanded.

7. **`/custom/modelconfigurationsetups/{id}`** (`custom_modelconfigurationsetups_id_get`): Same as above -- alias for the same operation.

8. **`/custom/modelconfigurations/{id}`** (`custom_modelconfigurations_id_get`): Get a single configuration with all nested data (full expansion). Deep query on `modelcatalog_model_configuration`.

9. **`/custom/models/standard_variable`** (`custom_models_standard_variable_get`): Find models by standard variable name. Query through versions -> input/output variable presentations -> standard variables.

10. **`/custom/datasetspecifications/{id}/datatransformations`** (`custom_datasetspecifications_id_datatransformations_get`): Get data transformations for a dataset specification. If no dedicated table exists, return empty array with a TODO comment.

11. **`/custom/datasetspecifications`** (`custom_datasetspecifications_get`): List dataset specifications with additional metadata. Query `modelcatalog_dataset_specification` with relationships.

12. **`/custom/configuration/{id}/inputs`** (`custom_configuration_id_inputs_get`): Get inputs for a configuration. Query through `modelcatalog_configuration_input` junction to `modelcatalog_dataset_specification`.

13. **`/user/login`** (`user_login_post`): This is actually an auth endpoint. The existing implementation likely forwards to Keycloak. For v2.0.0, implement as a pass-through that validates credentials against the auth server (or return 501 if the auth flow is handled externally). Check the existing FastAPI implementation for the actual behavior.

For each handler:
- Use `readClient.query(...)` (Apollo Client) for queries
- Apply the response mapper (`transformRow`/`transformList`) from `./mappers/response` to format results
- Accept the standard query params (username, label, page, per_page) where applicable. Filter by `user_id` when `username` is provided.
- Build GraphQL queries with appropriate nesting depth using `gql` from `@apollo/client/core`
- Handle the `id` parameter with `decodeURIComponent()` where present

Export as `customHandlers: Record<string, (req: any, reply: any) => Promise<void>>` keyed by operationId.
