Create a map from Hasura table name to its GraphQL field selection string. Derive these from the Hasura metadata (tables.yaml). For each modelcatalog table:

1. List all columns from the select_permissions (these are the scalar fields)
2. List all object_relationships (these are single FK joins -- include as nested selections)
3. List all array_relationships (these are multi-valued -- include as nested selections, but for junction tables go one level deeper to get the target entity)

```
readme

user_id
versions {
  id
  label
  description
}
authors {
  person {
    id
    label
  }
}
```

For junction table traversal: when an array_relationship points to a junction table (e.g., `modelcatalog_setup_author`), the field selection should include the junction's object_relationship to the target entity:
```
authors {
  person {
    id
    label
  }
}
```

For entity tables with direct FK relationships (e.g., `modelcatalog_software_version.software_id`), include the object_relationship:
```
software {
  id
  label
}
```

Export `FIELD_SELECTIONS: Record<string, string>` mapping table name to the field selection string (without outer braces).

Also export `getFieldSelection(tableName: string): string` that returns the selection for a table, falling back to just `id label` if not found.

Parse the actual tables.yaml file at build time or hardcode the selections based on reading the file now. Since the Hasura metadata is in a different repo (this mint repo), hardcode the selections based on the current tables.yaml content. Organize by table with comments.

Include field selections for ALL 16 entity tables and their relationship traversals. Junction tables themselves do not need standalone field selections (they are always accessed through entity relationships).

IMPORTANT: Include `user_id` in the field selections for entity tables that have it. This is needed for username-based filtering.
