    **Section 1: Summary classification counts**
    ```sql
    -- SECTION 1: Classification summary
    -- Shows how many model rows match ModelConfiguration, ModelConfigurationSetup, or are orphans
    SELECT
        CASE
            WHEN mc.id IS NOT NULL THEN 'ModelConfiguration'
            WHEN ms.id IS NOT NULL THEN 'ModelConfigurationSetup'
            ELSE 'ORPHAN'
        END AS classification,
        COUNT(*) AS row_count
    FROM model m
    LEFT JOIN modelcatalog_model_configuration mc ON mc.id = m.id
    LEFT JOIN modelcatalog_model_configuration_setup ms ON ms.id = m.id
    GROUP BY 1
    ORDER BY 1;
    ```

    **Section 2: Detailed row-by-row classification (the review report)**
    Per user decision: "Review report shows each old model row -> matched modelcatalog row -> classification type."
    ```sql
    -- SECTION 2: Detailed classification - every model row and its match
    -- Orphaned rows (no match in modelcatalog_* tables) keep null FK per user decision
    SELECT
        m.id AS model_id,
        m.name AS model_name,
        mc.id AS config_match,
        mc.label AS config_label,
        ms.id AS setup_match,
        ms.label AS setup_label,
        CASE
            WHEN mc.id IS NOT NULL THEN 'ModelConfiguration'
            WHEN ms.id IS NOT NULL THEN 'ModelConfigurationSetup'
            ELSE 'ORPHAN'
        END AS classification
    FROM model m
    LEFT JOIN modelcatalog_model_configuration mc ON mc.id = m.id
    LEFT JOIN modelcatalog_model_configuration_setup ms ON ms.id = m.id
    ORDER BY classification, m.id;
    ```

    **Section 3: Impact on execution and thread_model tables**
    ```sql
    -- SECTION 3: Execution table impact preview
    -- Shows how many execution rows will get each classification
    SELECT
        CASE
            WHEN mc.id IS NOT NULL THEN 'ModelConfiguration'
            WHEN ms.id IS NOT NULL THEN 'ModelConfigurationSetup'
            ELSE 'ORPHAN (null FK)'
        END AS classification,
        COUNT(*) AS execution_count
    FROM execution e
    JOIN model m ON m.id = e.model_id
    LEFT JOIN modelcatalog_model_configuration mc ON mc.id = m.id
    LEFT JOIN modelcatalog_model_configuration_setup ms ON ms.id = m.id
    GROUP BY 1
    ORDER BY 1;

    -- Same for thread_model
    SELECT
        CASE
            WHEN mc.id IS NOT NULL THEN 'ModelConfiguration'
            WHEN ms.id IS NOT NULL THEN 'ModelConfigurationSetup'
            ELSE 'ORPHAN (null FK)'
        END AS classification,
        COUNT(*) AS thread_model_count
    FROM thread_model tm
    JOIN model m ON m.id = tm.model_id
    LEFT JOIN modelcatalog_model_configuration mc ON mc.id = m.id
    LEFT JOIN modelcatalog_model_configuration_setup ms ON ms.id = m.id
    GROUP BY 1
    ORDER BY 1;
    ```

    **Section 4: Parameter binding orphan preview**
    ```sql
    -- SECTION 4: Parameter binding orphans (will be DELETED by migration 2)
    -- Review these counts before approving migration
    SELECT COUNT(*) AS orphaned_execution_param_bindings
    FROM execution_parameter_binding epb
    WHERE NOT EXISTS (
        SELECT 1 FROM modelcatalog_parameter mp WHERE mp.id = epb.model_parameter_id
    );

    SELECT COUNT(*) AS orphaned_thread_model_params
    FROM thread_model_parameter tmp
    WHERE NOT EXISTS (
        SELECT 1 FROM modelcatalog_parameter mp WHERE mp.id = tmp.model_parameter_id
    );
    ```

    **Migration 1: `1771200000000_fk_migration_classify`**

    `up.sql` must contain a single BEGIN/COMMIT transaction that:
    1. Adds two nullable FK columns to `execution`:
       - `modelcatalog_configuration_id TEXT REFERENCES modelcatalog_model_configuration(id) ON DELETE SET NULL`
       - `modelcatalog_setup_id TEXT REFERENCES modelcatalog_model_configuration_setup(id) ON DELETE SET NULL`
    2. Creates indexes: `idx_execution_mc_config` on `execution(modelcatalog_configuration_id)` and `idx_execution_mc_setup` on `execution(modelcatalog_setup_id)`
    3. Backfills `execution.modelcatalog_configuration_id` by matching `execution.model_id` against `modelcatalog_model_configuration.id` (WHERE EXISTS)
    4. Backfills `execution.modelcatalog_setup_id` for remaining unmatched rows against `modelcatalog_model_configuration_setup.id` (WHERE modelcatalog_configuration_id IS NULL AND EXISTS)
    5. Repeats steps 1-4 for `thread_model` with same column names and indexes (`idx_thread_model_mc_config`, `idx_thread_model_mc_setup`)
    6. Drops FK constraint `execution_model_id_fkey` from execution, then makes `execution.model_id` nullable (ALTER COLUMN model_id DROP NOT NULL)
    7. Drops FK constraint `thread_model_model_id_fkey` from thread_model, then makes `thread_model.model_id` nullable (ALTER COLUMN model_id DROP NOT NULL)

    Use exact constraint names from research: `execution_model_id_fkey`, `thread_model_model_id_fkey`.

    `down.sql` must reverse: drop new columns (CASCADE drops indexes and FK constraints), re-add old FK constraints, re-add NOT NULL on model_id. Wrap in BEGIN/COMMIT.

    **Migration 2: `1771200001000_fk_migration_parameter`**

    `up.sql` must contain a single BEGIN/COMMIT transaction that:
    1. Adds `modelcatalog_dataset_specification_id TEXT REFERENCES modelcatalog_dataset_specification(id) ON DELETE SET NULL` to `model_io`
    2. Creates index `idx_model_io_mc_ds` on `model_io(modelcatalog_dataset_specification_id)`
    3. Backfills `model_io.modelcatalog_dataset_specification_id` by matching `model_io.id` against `modelcatalog_dataset_specification.id` (WHERE EXISTS)
    4. Deletes orphaned rows from `execution_parameter_binding` where `model_parameter_id` NOT IN `modelcatalog_parameter.id` (DELETE FROM ... WHERE NOT EXISTS). These rows must be deleted (not nulled) because `model_parameter_id` is part of the primary key.
    5. Deletes orphaned rows from `thread_model_parameter` where `model_parameter_id` NOT IN `modelcatalog_parameter.id` (same reason -- PK includes model_parameter_id)
    6. Drops FK constraint `execution_parameter_binding_model_parameter_id_fkey`
    7. Drops FK constraint `thread_model_parameter_parameter_id_fkey`
    8. Adds new FK constraint `execution_parameter_binding_modelcatalog_parameter_fkey` on `execution_parameter_binding(model_parameter_id)` REFERENCES `modelcatalog_parameter(id)` ON DELETE SET NULL -- NOT DEFERRABLE (orphans already deleted)
    9. Adds new FK constraint `thread_model_parameter_modelcatalog_parameter_fkey` on `thread_model_parameter(model_parameter_id)` REFERENCES `modelcatalog_parameter(id)` ON DELETE SET NULL

    `down.sql` must reverse: drop new FK constraints, re-add old FK constraints, drop `modelcatalog_dataset_specification_id` column from model_io. Wrap in BEGIN/COMMIT. Note: deleted orphan rows cannot be restored in down.sql -- add a comment noting this.

    Important:

    - Add inline SQL comments explaining each section (classification, backfill, constraint changes)
