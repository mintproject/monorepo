<objective>
Default the `type` column to `resourceConfig.typeUri` when creating resources via POST.

Purpose: The `toHasuraInput` mapper intentionally skips the `type` field from the request body (line 93-94 of request.ts) since the API-level type value (e.g., `["Model"]`) is a short name, not the full URI stored in Hasura. But the `create` method in service.ts never sets the `type` column, so newly created rows have no type. This fix adds a single line in the create method to always set `input['type'] = resourceConfig.typeUri` after `toHasuraInput` returns.

Output: Updated service.ts with default type assignment, plus a test validating the behavior.
</objective>
