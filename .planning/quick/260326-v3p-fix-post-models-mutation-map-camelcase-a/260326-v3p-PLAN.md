<objective>
Fix POST /models (and all resource POST/PUT endpoints) to not send unknown fields to Hasura.

Currently, `toHasuraInput` blindly converts ALL non-relationship camelCase body fields to snake_case and sends them to Hasura. This causes errors when:
1. Frontend sends `shortDescription` (a softwareversions column, not a software column) -- camelToSnake produces `short_description` which doesn't exist on `modelcatalog_software_insert_input`
2. Frontend sends `hasModelCategory` (a relationship on softwareversions, not on models) -- not in models' relationship map, so falls through to scalar processing and becomes `has_model_category`

Fix: Make `toHasuraInput` validate snake_case field names against the known columns from `FIELD_SELECTIONS` for the target table. Unknown fields are silently dropped.

Purpose: Unblock model creation from the frontend UI
Output: Working POST /models that ignores unknown/inapplicable fields
</objective>
